1. Установите Git на свой компьютер:
https://git-scm.com/
git config --global user.name (тут придумайте никнейм)
git config --global user.email (ваша почта@домен)

2. Создайте папку и инициализируйте репозиторий в выбранной папке:
git init

3. Создайте текстовый файл с произвольным содержимым.

4. Зафиксируйте изменения данного файла в Git:
git add . (или git add (название файла))
git commit -m 'изменения'

5. Измените файл снова.

6. Снова зафиксируйте изменения данного файла:
git commit -m 'изменения'

7. Свяжите локальный репозиторий с удалённым на Github:
на гитхабе создаёте ПУБЛИЧНЫЙ репозиторий и подключаете через
git remote (ссылка на репозиторий).git
ЕСЛИ НЕ СРАБОТАЕТ:
git remote set-url origin (ссылка на репозиторий).git

8. Опубликуйте ваши изменения:
git push origin main (или master)

9. Создайте и переключитесь на новую ветку feature:
git branch feature
git checkout feature

10. Добавьте новый текстовый файл, зафиксируйте в репозитории:
git add . (или git add (название файла))
git commit -m 'изменения'
git push???

11. Добавьте третий файл. Зафиксируйте:
аналогично 10

12. Опубликуйте ветку:
push origin -u feature

13. Откатите коммит по добавлению третьего файла:
git revert (индекс операции) ((набор букв и цифр после добавления/пуша/коммита, точно не помню)

14. Опубликуйте текущее состояние репозитория:
git push

15. Посмотрите историю всех комитов:
git log

16. Создайте мёрж-реквест на слияние ветки feature в ветку main:
Вот я тут вроде жидко обосрался, вроде должно быть:
git checkout main (или master)
git merge feature

ПО ВСЕМ ВОПРОСАМ ПИШИТЕ МНЕ (лёхехехе) помогу как смогу
